Fase 1 — Predicción de Prediabetes y Diabetes (BRFSS 2015)

1 Resumen ejecutivo

Este informe documenta el desarrollo de un enfoque de clasificación probabilística para enfermedad glucémica utilizando el dataset BRFSS 2015 con etiqueta Diabetes_012 (0 = sano, 1 = prediabetes, 2 = diabetes). Se implementó un enfoque jerárquico con dos regresiones logísticas (glm binomial) para producir probabilidades coherentes de tres clases (sano / prediabetes / diabetes).

El sistema se empaquetó en una API REST mediante plumber, documentada con Swagger, con un contrato de salida que reporta: - Probabilidades por clase. - Clase final basada en umbrales (criterio de Youden).

2 Objetivo y alcance

Objetivo: predecir prediabetes o diabetes a partir de indicadores clínicos y de estilo de vida presentes en BRFSS 2015, entregando:

  1. Probabilidades:
    • p_diabetes
    • p_prediabetes
    • p_healthy
  2. Una clase final (final_class) definida por umbrales (Youden).

Alcance y supuestos:
- Modelo explicativo-predictivo (asociación; no causalidad).
- Datos auto-reportados: se asume ruido de medición.
- Se implementa glm puro (sin regularización) por consistencia con lo cubierto en el curso.
- Variables ordinales se tratan como numéricas (asunción de “efecto lineal por unidad” en el logit).

3 Datos y variable objetivo

El dataset utilizado contiene la variable Diabetes_012:

  • 0: no diabetes (incluye “solo durante embarazo”, según la codificación del dataset).
  • 1: prediabetes
  • 2: diabetes

La decisión de trabajo es usar Diabetes_012 (0/1/2) y derivar objetivos binarios internos para entrenamiento.

4 Metodología y definición del modelo

4.1 Enfoque jerárquico (Modelo A + Modelo B)

Se usan dos modelos logísticos:

Modelo A (diabetes):

  • Objetivo: Y_A = 1{Diabetes_012 = 2}, 0{Diabetes_012 in {0,1}}

  • Salida: p_diabetes = P(D=2)

Modelo B (prediabetes condicional):

  • Entrenado solo en registros con Diabetes_012 in {0,1}

  • Objetivo: Y_B = 1{Diabetes_012 = 1}, 0{Diabetes_012 = 0}

  • Salida: p_pred_cond = P(D=1 | D != 2)

Se derivan probabilidades coherentes de 3 clases:

  • p_prediabetes = (1 - p_diabetes) * p_pred_cond

  • p_healthy = 1 - p_diabetes - p_prediabetes

4.2 Umbral de decisión: Youden

Los umbrales de clasificación se obtienen con el criterio de Youden (maximiza sensibilidad + especificidad − 1). Se guardan dentro del bundle para usar los mismos umbrales en API y en reporte.

7 Interpretación del GLM: Odds Ratios (OR) e IC95%

Se reportan OR e IC95% (Wald). Interpretación:

  • Para variables binarias 0/1, el OR representa el cambio relativo en odds al pasar de 0 a 1.

  • Para variables ordinales tratadas como numéricas, el OR se interpreta por incremento de 1 unidad (supuesto fuerte; debe destacarse).

8 Contrato del API (Plumber + Swagger)

La API expone endpoints:

  • GET /health — verificación de estado.

  • POST /predict — recibe un JSON con los predictores del modelo y devuelve probabilidades y clase final.

Salida esperada:

  • probabilities: {p_diabetes, p_prediabetes, p_healthy}

  • final_class: etiqueta final basada en umbrales (Youden)

9 Limitaciones y riesgos

  • Auto-reporte: variables y diagnóstico pueden tener error sistemático.

  • Observacional: el GLM captura asociación, no causalidad.
